Projet de dashboard d'un fast food par ABONDO Ian et PERRIER Romain

L’objectif : Centraliser et orchestrer les flux de commandes provenant de multiples plateformes (UberEats, Deliveroo, Just Eat) sur un seul écran pour optimiser le temps de préparation et réduire les erreurs de livraison.

Tâches :

  Tâche 1 : Priorisation automatique des commandes
    Action : Le système calcule l'urgence selon le temps restant avant l'arrivée du livreur.
    Interaction : Affichage d'un code couleur (Vert/Orange/Rouge clignotant) pour attirer l'œil sur les commandes prioritaires.

  Tâche 2 : Validation de la préparation
    Action : Cocher les items d'une commande un par un pour éviter les oublis (ex: boissons, sauces).
    Contrainte d'interaction : Utilisation de gestes simples (swipe) pour limiter les contacts précis avec l'écran (mains sales).

  Tâche 3 : Gestion des incidents (Rupture de stock)
    Action : Signaler en temps réel qu'un ingrédient n'est plus disponible.
    Interaction : Menu "Accès rapide" pour éviter de naviguer dans des sous-menus complexes en plein rush.
  Tâche 4 : Coordination du Flux Livreurs
    Action : Assurer la transition fluide entre la cuisine et le coursier.
    Problème d'interaction : Identifier le bon sac parmi des dizaines. L'interface doit générer un identifiant visuel géant pour que le livreur et le manager se comprennent sans crier.
    
  Tâche 5 : Ajustement de la Capacité 
    Action : Réguler l'arrivée de nouvelles commandes si la cuisine est saturée.
    Interaction : Un curseur de "Niveau de Stress" (de 1 à 5). Plus le manager glisse vers le haut, plus le système augmente automatiquement les délais sur toutes les plateformes partenaires pour donner de l'air à l'équipe.
